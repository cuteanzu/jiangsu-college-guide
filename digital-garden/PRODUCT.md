# Product

## Register

brand

## Users

个人创意展示 + 数据可视化实验场。面向招聘方、同行、以及自己的探索欲。用户带着好奇心进入，像逛一座数字园林，每个入口通向不同的数据景观。

## Product Purpose

多模块数据可视化平台，以"数字园林"为隐喻组织多个独立的数据世界：
- 作品馆（作品集、博客、关于 — 待定中式命名）
- 江苏高校地图（3D 地图 + 羊皮地图风格 + 高校分布与特色）
- 未来可扩展更多数据景观模块

核心体验：登录即入园，导航即穿行，每次访问都是一场游园。

## Brand Personality

三个词：**诗意的、画卷式的、有手工感的**

- 不是冷冰冰的数据面板，是匠人手绘的图卷
- 不是扁平的单页，是层层递进的画卷空间（如《清明上河图》——横向展开、步移景异）
- 纯中国古典美学，排除日本和风元素
- 文化参照系：唐宋山水、明清小说版画、古籍善本、羊皮地图、篆刻印章、水墨设色
- 神话与文学：二郎神、山海经、西游记等可作为角色/场景设计的形象来源
- 古文字入画：篆书、隶书、楷书作为视觉元素，而非仅仅字体选择
- 不堆砌灯笼、祥云、毛笔字——东方但不刻板
- 有古意但不陈旧——像古画的数字重生，能动、能交互
- 材料感优先：纸纹、墨色、绢本、朱砂印、石青、赭石——矿物颜料与植物染料色域

## Anti-references

- 传统数据大屏（深蓝底 + 荧光边框 + 数字跳动）= 完全排斥
- GIS 学术风（灰色底图 + 彩色热力 + 图例面板）
- 政府公示平台风格
- 赛博朋克霓虹灯
- 西方极简 Swiss/Helvetica 风
- 二次元动漫风
- 日本和风 / 侘寂美学（与本项目的中国古典定位冲突）
- Bootstrap 套模板感
- SaaS 产品 dashboard（白底 + 蓝色 sidebar + 表格）

## Design Principles

1. **画卷式展开，非页面跳转。** 如《清明上河图》横向舒展、移步换景——信息不是切换 tab，是在画卷中游走。一屏一景，景景相连。
2. **手工材料感。** 纸、墨、绢、印泥、羊皮、石青、赭石——用数字媒介还原中国古典材料的温度和质感。OKLCH 取色偏向矿物颜料、植物染料、古建彩画色域。
3. **古画新生。** 中国绘画传统（界画楼阁、山水皴法、人物白描、工笔设色）用现代技术重新表达——Three.js 渲染、着色器、粒子系统。看起来像古画，但它能动。
4. **文字即风景。** 篆隶楷行草不只是字体——是画面元素。匾额、题跋、印章、碑刻，文字作为空间中的视觉存在。
5. **藏大于露。** 障景、曲径、半掩——如园林漏窗，如手卷逐段展开。给用户探索的乐趣，不一次性摊开所有信息。

## Accessibility & Inclusion

- `prefers-reduced-motion` 已通过 seasonalfx 的 `respectReducedMotion: true` 响应
- 地图类模块需关注色盲友好：不只依赖颜色区分数据，加纹理/形状双通道编码
- 目标 WCAG AA，但东方美学天然偏柔和低对比——需在诗意与可读性之间找到平衡点，不做非黑即白的取舍
